How it works

Paste a bug, feature idea, PRD, or screenshot. brocode picks the mode and runs the right agents automatically.

🔴 Investigate — Bug / Incident
🔵 Spec — Feature / Design
🟢 Develop — Implement
🟣 Review — PR / MR
1
TPM initialises run log
Creates investigation ID, logs all stage transitions throughout
2
Tech Lead dispatches scoped sub-agents sub-agents parallel
Backend / Frontend / Mobile spawned based on which layer the bug is in. Read real repos. Debate in swe-debate.md.
3
SRE assesses blast radius parallel
Runs in parallel with Tech Lead's team. Asks questions about topology and deploy window.
4
Engineering BR loop max 3 rounds
Fresh BR sub-agent per round challenges root cause and fix. Fresh producer sub-agent revises. Escalates to you after 3 rounds.
6
engineering-spec.md + tasks.md
Confirmed root cause + exact fix + failing test + ops rollback + domain-scoped task list for implementation.

The team

12 agents. Each owns a specific domain. Each reads real code. Each challenges the others.

📋

TPM

claude-sonnet
Program orchestrator. Runs start to finish. Logs every transition, every BR round, every blocker. Prints live progress. Detects stalls. Invokes superpowers:using-superpowers at session start.
Cross-cutting
🎯

Product Manager

claude-opus
Turns any input into requirements — full user journeys for end users, ops/admin, and support teams. Closes ambiguity gaps. References competitors. Owns UX flows (section 15).
Product
🔬

Product Bar Raiser

claude-opus
Principal PM. Challenges every untestable AC, missing persona, and unresolved assumption. Web-searches competitors to verify claims. Hard gate — engineering blocked until PM approved.
Product Gate
🤝

Tech Lead

claude-sonnet
Owns the engineering team: Backend, Frontend, Mobile, SRE. Scopes which sub-agents to dispatch. Runs the parallel debate. Synthesises 3 implementation options. Bridges product and engineering tracks.
Engineering
⚙️

Backend Engineer

claude-sonnet
Sub-agent. Reads real backend code — traces request lifecycle, checks DB schema, finds existing patterns. Owns APIs, data models, auth, queues. Uses superpowers:systematic-debugging when stuck.
EngineeringSub-agent
🖥️

Frontend Engineer

claude-sonnet
Sub-agent. Reads real frontend code — components, state, API client. Owns UI, state management, SSR/CSR. Challenges Backend on API usability and round-trip count. Uses superpowers:systematic-debugging when stuck.
EngineeringSub-agent
📱

Mobile Engineer

claude-sonnet
Sub-agent. Reads real mobile code — iOS, Android, RN, Flutter. Challenges Backend on offline behavior, payload size, battery. App store compliance awareness. Uses superpowers:systematic-debugging when stuck.
EngineeringSub-agent
🚨

SRE

claude-sonnet
Tech Lead's team. Ops plan — observability, alerts, blast radius, executable rollback, runbooks, pre-deploy checklist. Asks SWE about topology first. Runs parallel with QA.
Engineering
🧪

QA Engineer

claude-sonnet
Full test matrix with real test code — unit, integration, E2E, load, security. Every AC covered. No TODOs. Asks Tech Lead about edge cases. Runs parallel with SRE.
Engineering
⚖️

Engineering Bar Raiser

claude-opus
Principal Engineer. Challenges all 4 engineering artifacts — cross-artifact consistency check. Fresh sub-agent per review round. Writes engineering-spec.md + tasks.md when all approved.
Engineering Gate

Two Bar Raisers

Adversarial reviewers. Their challenges are blockers, not suggestions. Fresh sub-agent per round — no context drift. Max 3 rounds, then escalates to you with a specific question.

Product Track

Product Bar Raiser

Principal PM / Head of Product. Hard gate — engineering cannot start until PM approved.

  • Challenges every AC that is untestable or unmeasurable
  • Verifies all personas: end user, ops/admin, support team
  • Checks every journey has error states and recovery actions
  • Web-searches competitor references — verifies claims against real behavior
  • Challenges designs for missing error states and edge case flows
  • Verifies ops team can actually manage this feature post-launch
Fresh BR sub-agent per round → challenged → fresh producer sub-agent revises → max 3 rounds
Engineering Track

Engineering Bar Raiser

Principal Engineer. Reviews all 4 engineering artifacts. Writes engineering-spec.md + tasks.md when all approved.

  • Challenges Tech Lead options for vague tradeoffs or inconsistency with design
  • Challenges SRE rollback plans that are theoretical rather than executable
  • Challenges QA for ACs without tests or TODOs instead of real test code
  • Cross-checks all artifacts for mutual consistency
Fresh BR sub-agent per round → challenged → fresh producer sub-agent revises → max 3 rounds

Register your repos

Engineer sub-agents read real code before proposing anything. Any domain, multiple repos per domain — run once per machine.

1

Run the setup command

Enter repos as domain: /path lines. brocode validates each path exists, then asks for description, labels, and tags per repo.

2

Saved to ~/.brocode/repos.json

User-level — shared across all projects on this machine. Re-run anytime to update.

3

Agents orient from metadata, then read code

Each agent reads description, labels, and tags before exploring files — so they know what the repo does before diving in.

4

Domain → agent mapping

backend → Backend Engineer · mobile → Mobile · web/fullstack → Frontend · terraform/infra/sre → SRE · qa → QA

/brocode:brocode repos

Input formats

Paste, attach, or link. Never silently fails — if an MCP is missing, brocode tells you exactly what to do.

💬

Plain text

Describe the bug or feature — as much or as little detail as you have

🖼️

Image / Screenshot

Attach Figma exports, error screens, or mockups — PM analyzes via vision

📄

Google Doc

Paste a link — fetched automatically via Google Drive MCP

Requires Google Drive MCP
📝

Notion / Wiki

No MCP by default — brocode asks you to paste the content directly

🎨

Figma

No Figma MCP by default — brocode asks you to export as PNG and attach

Install

Four commands. Copy, paste, run. Restart Claude Code — type /broc and select /brocode:brocode from autocomplete.

1

Clone brocode

git clone
  https://github.com/im-adarsh/brocode.git
  ~/brocode
Clone once. Update anytime with cd ~/brocode && git pull
2

Register with Claude Code

claude plugin marketplace add
  ~/brocode
Points Claude Code at your local clone
3

Install brocode + superpowers

claude plugin install brocode@brocode-local --scope user

claude plugin install
  superpowers@claude-plugins-official
  --scope user
superpowers required for /brocode:brocode develop and /brocode:brocode review · Migrating from old install? claude plugin uninstall sdlc@brocode-local first
4

Restart Claude Code → start using it
