How it works

Paste a bug, feature idea, PRD, or screenshot. brocode picks the mode and runs the right agents automatically.

3
SRE assesses blast radius parallel

Checks for systemic causes beyond this instance. Converses with Tech Lead via eng thread.

The team

12 agents. Each owns a specific domain. Each reads real code. Each challenges the others.

📋

TPM

claude-sonnet
Program orchestrator. Runs start to finish. Logs every transition, every BR round, every blocker. Prints live progress. Detects stalls. Invokes superpowers:using-superpowers at session start.
Cross-cutting
🎯

Product Manager

claude-opus
Turns any input into requirements — full user journeys for end users, ops/admin, and support teams. Closes ambiguity gaps. References competitors. Converses with Designer.
Product
🎨

Designer

claude-sonnet
Designs API contracts, all user flows (happy, error, empty, loading), ops interface, and support interface. Converses with PM. Answers engineering questions during the engineering track.
Product
🔬

Product Bar Raiser

claude-opus
Principal PM. Challenges every untestable AC, missing persona, and unresolved assumption. Web-searches competitors to verify claims. Hard gate — engineering blocked until both PM + Designer approved.
Product Gate
🤝

Tech Lead

claude-sonnet
Owns the engineering team: Backend, Frontend, Mobile, SRE. Scopes which sub-agents to dispatch. Runs the parallel debate. Synthesises 3 implementation options. Bridges product and engineering tracks.
Engineering
⚙️

Backend Engineer

claude-sonnet
Sub-agent. Reads real backend code — traces request lifecycle, checks DB schema, finds existing patterns. Owns APIs, data models, auth, queues. Uses superpowers:systematic-debugging when stuck.
EngineeringSub-agent
🖥️

Frontend Engineer

claude-sonnet
Sub-agent. Reads real frontend code — components, state, API client. Owns UI, state management, SSR/CSR. Challenges Backend on API usability and round-trip count. Uses superpowers:systematic-debugging when stuck.
EngineeringSub-agent
📱

Mobile Engineer

claude-sonnet
Sub-agent. Reads real mobile code — iOS, Android, RN, Flutter. Challenges Backend on offline behavior, payload size, battery. App store compliance awareness. Uses superpowers:systematic-debugging when stuck.
EngineeringSub-agent
🏗️

Staff SWE

claude-opus
System-level architectural review. Converses with Tech Lead before forming opinions. Identifies failure modes, hidden coupling, scalability cliffs. Peer — not manager — of Tech Lead. Joint recommendation.
Engineering
🚨

SRE

claude-sonnet
Tech Lead's team. Ops plan — observability, alerts, blast radius, executable rollback, runbooks, pre-deploy checklist. Asks SWE about topology first. Runs parallel with QA.
Engineering
🧪

QA Engineer

claude-sonnet
Full test matrix with real test code — unit, integration, E2E, load, security. Every AC covered. No TODOs. Asks Tech Lead about edge cases. Runs parallel with SRE.
Engineering
⚖️

Engineering Bar Raiser

claude-opus
Principal Engineer. Challenges all 4 engineering artifacts — cross-artifact consistency check. Fresh sub-agent per review round. Writes 08-final-spec.md + 09-tasks.md when all approved.
Engineering Gate

Two Bar Raisers

Adversarial reviewers. Their challenges are blockers, not suggestions. Fresh sub-agent per round — no context drift. Max 3 rounds, then escalates to you with a specific question.

Product Track

Product Bar Raiser

Principal PM / Head of Product. Hard gate — engineering cannot start until PM and Designer are both approved.

  • Challenges every AC that is untestable or unmeasurable
  • Verifies all personas: end user, ops/admin, support team
  • Checks every journey has error states and recovery actions
  • Web-searches competitor references — verifies claims against real behavior
  • Challenges API contracts for missing error cases
  • Verifies ops team can actually manage this feature post-launch
Fresh BR sub-agent per round → challenged → fresh producer sub-agent revises → max 3 rounds
Engineering Track

Engineering Bar Raiser

Principal Engineer. Reviews all 4 engineering artifacts. Writes 08-final-spec.md + 09-tasks.md when all approved.

  • Challenges Tech Lead options for vague tradeoffs or inconsistency with design
  • Challenges Staff SWE concerns without codebase evidence
  • Challenges SRE rollback plans that are theoretical rather than executable
  • Challenges QA for ACs without tests or TODOs instead of real test code
  • Cross-checks all 4 artifacts for mutual consistency
  • Tech Lead recommendation must match Staff SWE architecture — flags divergence
Fresh BR sub-agent per round → challenged → fresh producer sub-agent revises → max 3 rounds

Input formats

Paste, attach, or link. Never silently fails — if an MCP is missing, brocode tells you exactly what to do.

💬

Plain text

Describe the bug or feature — as much or as little detail as you have

🖼️

Image / Screenshot

Attach Figma exports, error screens, or mockups — PM and Designer analyze via vision

📄

Google Doc

Paste a link — fetched automatically via Google Drive MCP

Requires Google Drive MCP
📝

Notion / Wiki

No MCP by default — brocode asks you to paste the content directly

🎨

Figma

No Figma MCP by default — brocode asks you to export as PNG and attach

claude plugin marketplace add
